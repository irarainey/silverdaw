# Application resources

Static files referenced at runtime by the Electron main process and at
packaging time by electron-builder. Kept **outside** the build output
so they're addressable via `app.getAppPath()` in `pnpm dev` and via
`process.resourcesPath` in a packaged install, without needing to be
re-emitted by Vite.

## `icons/`

Generated by [`scripts/Build-InstallerArt.py`](../../scripts/Build-InstallerArt.py)
from `frontend/resources/icons/256x256.png` (the source Silverdaw logo).

| File                                                              | Purpose                                                                                                                                                                                                                  |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `icon.ico`                                                        | Windows window / taskbar / alt-tab icon. Wired in `frontend/src/main/index.ts` via `join(app.getAppPath(), 'resources', 'icons', 'icon.ico')`. Defensive lookup: if missing, Electron falls back to its default icon and `main.log` notes the absence — the app still starts. |
| `silverdaw-file.ico`                                              | File-type icon for `.silverdaw` documents (white page with folded corner + Silverdaw logo), used by Explorer / the taskbar at classic icon sizes. |
| `16x16.png` … `256x256.png`                                       | Per-size PNGs. `256x256.png` is also the input to the packaging-art generator script. Picked up by electron-builder when packaging.                                       |

## `appx/`

MSIX/AppX package assets, also generated by
[`scripts/Build-InstallerArt.py`](../../scripts/Build-InstallerArt.py) and copied
into the package's `assets/` folder by electron-builder at packaging time.

| File(s)                                                                                  | Purpose                                                                                                                                                              |
| ---------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `StoreLogo`, `Square44x44Logo`, `Square150x150Logo`, `Wide310x150Logo` (+ `.scale-*`)    | Tile logos on an opaque `#F3F3F3` plate (matching the light App Installer dialog), with per-DPI scale variants. `StoreLogo` is what the App Installer dialog shows. |
| `Square44x44Logo.targetsize-*` (plain / `_altform-unplated` / `_altform-lightunplated`)  | Unplated icons Windows themes for the taskbar, Start and jump lists (dark / light backgrounds).                                                                     |
| `silverdaw-file.png`, `silverdaw-file.targetsize-*`                                      | The `.silverdaw` document icon as PNGs, referenced by the file-type `<uap:Logo>`.                                                                                    |

## `appx-extensions.xml`

A custom AppxManifest `<Extensions>` fragment injected via
`appx.customExtensionsPath` in [`../electron-builder.yml`](../electron-builder.yml).
It registers the `.silverdaw` file type with a friendly name, a tooltip, and the
document `<uap:Logo>` (`assets\silverdaw-file.png`) — richer than
electron-builder's built-in `fileAssociations`, which emits only a bare
association.

## Regenerating

Regenerate the icon set and the appx assets by running:

```powershell
pip install Pillow
python scripts/Build-InstallerArt.py
```

The outputs are checked into git so the normal release build doesn't need
Python on the PATH.
